# Références — Jour 5 : Prompt Engineering

## Références conceptuelles

- Documentation fournisseur sur les bonnes pratiques de prompting.
- Guides de sortie structurée et validation JSON.
- Littérature sur les risques de prompt injection.
- Articles d'ingénierie sur l'évaluation de prompts.
- Documentation Python sur `json`, `dataclasses` et tests unitaires.

## Lectures recommandées

1. Prompt design pour les tâches de classification.
2. Few-shot prompting et sélection d'exemples.
3. Prompt injection et séparation instruction / données.
4. Évaluation de prompts et jeux de tests.
5. Structured outputs et schémas de validation.

## À retenir

Un prompt professionnel doit être :

- clair ;
- court ;
- testable ;
- versionné ;
- compatible avec la validation applicative ;
- intégré à une architecture logicielle.

## Préparation au Jour 6

Relire les notions suivantes :

- JSON valide ;
- schéma de données ;
- validation d'entrée et de sortie ;
- différence entre texte libre et sortie structurée ;
- séparation entre raisonnement du modèle et action d'un système.
