# Objectifs d’apprentissage

## Objectifs principaux

À la fin de cette journée, l’apprenant doit être capable de :

- expliquer pourquoi un client Python LLM doit être isolé du reste de l’application ;
- concevoir une interface minimale pour envoyer une requête texte à un modèle ;
- gérer une configuration par variables d’environnement ;
- distinguer un transport réel, un transport mock et une logique métier ;
- récupérer une réponse texte ;
- valider une réponse structurée ;
- implémenter une gestion d’erreurs lisible ;
- écrire des tests unitaires sans appeler l’API externe ;
- préparer le mini-projet de la Semaine 1.

## Compétences AI Engineering

Cette journée introduit des pratiques essentielles pour un AI Backend Engineer :

- séparation des responsabilités ;
- configuration externalisée ;
- testabilité ;
- observabilité minimale ;
- robustesse ;
- interfaces stables ;
- préparation à l’intégration API, agent et production.

## Critères de validation

La journée est réussie si l’apprenant peut :

- créer une instance de client ;
- envoyer un prompt ;
- obtenir une sortie déterministe en mode mock ;
- parser une sortie JSON conforme à un schéma ;
- expliquer les limites d’un wrapper trop fin autour d’un SDK ;
- écrire un test automatisé pour le comportement attendu.
