Ce fichier sert de support de revue pour le formateur et pour le notebook enseignant.

## Erreurs fréquentes à signaler

1. **Confondre prototype et code de production**  
   Un prototype démontre une idée. Un livrable de production doit être testable, observable et maintenable.

2. **Oublier les cas limites**  
   Les entrées vides, champs manquants, types incorrects et erreurs de dépendance doivent être anticipés.

3. **Sur-optimiser trop tôt**  
   Avant d’optimiser, il faut disposer d’une baseline et d’un critère de mesure.

4. **Masquer les erreurs**  
   Une exception silencieuse rend le diagnostic difficile. Préférer des messages explicites et exploitables.

## Feedback type à donner aux apprenants

- Clarifier les hypothèses avant d’écrire le code.
- Produire une solution simple, testable et lisible.
- Documenter les décisions importantes.
- Justifier les compromis techniques.
